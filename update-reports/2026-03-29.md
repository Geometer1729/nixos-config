## Config Changes Required

### **FIXED: telescope.nvim / plenary.nvim** (`modules/home/nvim/nixvim.nix`)
nixpkgs changed how neovim resolves Lua modules (commits: `neovim: rely exclusively on evaltime deps`, `neovim: add package.path only when necessary`, `neovim: now include nixpkgs wrapper generated lua`). The new neovim uses the luajit-env for `require()` resolution instead of the runtimepath, so plugins that `require()` across plugin boundaries need to be declared as `extraLuaPackages`.

**Symptom**: `module 'plenary.strings' not found` when loading telescope
**Fix**: Added `extraLuaPackages = ps: with ps; [ plenary-nvim ];` to nixvim.nix
